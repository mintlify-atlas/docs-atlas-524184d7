The Statistics API surfaces the authenticated user's personal records (PRs) — the maximum weight ever lifted for every exercise they have trained. PR data is aggregated directly in SQL across all completed sets (`completada = 1`) and returned ordered by `peso_maximo DESC`. This endpoint powers the progress charts on the Blackterz frontend dashboard.

---

## GET /api/estadisticas/prs

Returns the personal record — the maximum weight ever lifted — for every exercise the authenticated user has logged at least one completed set on. Only sets where `sesion_series.completada = 1` are considered. Results are ordered by `peso_maximo DESC`.

**Auth required:** Yes — `Authorization: Bearer <token>`

The query joins four tables: `sesiones_entrenamiento` → `sesion_ejercicios` → `sesion_series` → `ejercicios`. It groups by exercise and aggregates with `MAX(peso)`.

<Tip>
  Poll this endpoint after every completed workout session to keep the frontend progress charts up to date.
</Tip>

### Request

No query parameters or request body. The `usuario_id` is extracted directly from the JWT by the `verificarToken` middleware.

```bash
curl -X GET https://api.blackterzgym.com/api/estadisticas/prs \
  -H "Authorization: Bearer <your_token>"
```

### Response

<ResponseField name="status" type="string" required>
  `"ok"` on success.
</ResponseField>

<ResponseField name="data" type="array" required>
  Array of personal record objects. One entry per exercise where the user has at least one completed set. Empty array `[]` if the user has no workout history.

  <Expandable title="Personal record object fields">
    <ResponseField name="ejercicio_id" type="integer">
      ID of the exercise, matching the `ejercicios.id` primary key.
    </ResponseField>
    <ResponseField name="ejercicio_nombre" type="string">
      Display name of the exercise, e.g. `"Peso muerto"`.
    </ResponseField>
    <ResponseField name="peso_maximo" type="number">
      Maximum weight in kilograms recorded across all completed sets for this exercise.
    </ResponseField>
    <ResponseField name="reps_maximas" type="integer">
      Maximum repetitions recorded in a single completed set for this exercise.
    </ResponseField>
    <ResponseField name="veces_entrenado" type="integer">
      Number of distinct `sesion_ejercicio` entries (unique exercise appearances across sessions) for this exercise.
    </ResponseField>
    <ResponseField name="total_series_completadas" type="integer">
      Total count of completed sets (`completada = 1`) logged for this exercise across all sessions.
    </ResponseField>
  </Expandable>
</ResponseField>

<Accordion title="500 Internal Server Error">
  Returned when the SQL query fails.

  ```json
    "message": "Error al obtener los récords personales"
  }
  ```
</Accordion>

<Accordion title="401 Unauthorized">
  Returned when no token or an invalid/expired token is supplied. Raised by the `verificarToken` middleware before the controller runs.

  ```json
  {
    "status": "error",
    "message": "Token no válido o expirado"
  }
  ```

## GET /api/health

Verifies that the API server is running and that the MySQL database connection is healthy. Executes a lightweight `SELECT 1` query against the pool. No authentication required.

**Auth required:** None

### Response

<Tabs>
  <Tab title="200 — Healthy">
    <ResponseField name="status" type="string">
      `"ok"`
    </ResponseField>
    <ResponseField name="db" type="string">
      `"conectada"` — the `SELECT 1` completed successfully.
    </ResponseField>

    ```json
    {
      "status": "ok",
      "db": "conectada"
    }
    ```
  </Tab>

  <Tab title="500 — Unhealthy">
    When the database connection throws an error, the health route calls `next(error)` and the global error handler in `server.js` responds with a generic 500.

    <ResponseField name="status" type="string">
      `"error"`
    </ResponseField>
    <ResponseField name="message" type="string">
      `"Error interno del servidor"`
    </ResponseField>

    ```json
    {
      "status": "error",
      "message": "Error interno del servidor"
    }
    ```
  </Tab>
</Tabs>

### Curl example

<CodeGroup>
```bash cURL
curl -X GET https://api.blackterzgym.com/api/health
```

```javascript JavaScript (fetch)
const res = await fetch('/api/health');
const { status, db } = await res.json();
if (db !== 'conectada') {
  console.error('Database is unreachable');
}
```
</CodeGroup>
